# 功能特性详解

## 🎨 UI/UX 特性

### 1. 简约设计风格

- **极简色彩方案**: 使用黑白灰为主的简约配色
- **清晰的视觉层次**: 通过字体大小、间距和颜色深浅建立清晰的视觉层次
- **舒适的阅读体验**: 精心选择的行高、字间距和段落间距
- **微妙的交互反馈**: 平滑的过渡动画和悬停效果

### 2. 深色模式

- **一键切换**: 点击导航栏的太阳/月亮图标即可切换
- **CSS 变量实现**: 所有颜色通过 CSS 变量定义，易于自定义
- **平滑过渡**: 主题切换时颜色平滑过渡
- **状态持久化**: 使用 localStorage 保存用户偏好

#### 颜色对比

| 元素     | 浅色模式 | 深色模式 |
| -------- | -------- | -------- |
| 背景     | #FFFFFF  | #1A1A1A  |
| 次要背景 | #F8F8F8  | #2A2A2A  |
| 主文字   | #333333  | #EFEFEF  |
| 次要文字 | #666666  | #AAAAAA  |

### 3. 响应式布局

#### 断点设计

- **移动端** (< 640px): 单列布局，简化导航
- **平板** (640px - 1024px): 双列布局，保持可读性
- **桌面** (> 1024px): 三列布局，充分利用空间

#### 响应式特性

- 导航栏在移动端自动折叠
- 瀑布流列数自动调整
- 文字大小和间距适配不同屏幕
- 触摸友好的按钮和链接

### 4. 瀑布流布局

#### 实现原理

```typescript
// 自动计算列数
const columns = screenWidth < 640 ? 1 : screenWidth < 1024 ? 2 : 3;

// 轮流分配内容到各列
items.forEach((item, index) => {
  columnItems[index % columns].push(item);
});
```

#### 特点

- 自适应列数
- 均匀分布内容
- 保持内容顺序
- 响应式调整

## 🌍 国际化（i18n）

### 支持的语言

- 🇨🇳 简体中文 (zh)
- 🇺🇸 English (en)

### 实现方式

- **next-intl**: 使用业界标准的 i18n 库
- **路由级别**: 通过 `/zh` 和 `/en` 路径区分语言
- **动态切换**: 一键切换语言，无需刷新页面
- **翻译文件**: JSON 格式的翻译文件，易于管理

### 添加新语言

1. 创建翻译文件: `messages/ja.json`
2. 更新配置: 在 `i18n.ts` 添加 `"ja"`
3. 所有页面自动支持新语言

## 🧩 组件架构

### 核心组件

#### 1. Navigation (导航栏)

- **位置**: 固定在顶部
- **特性**:
  - 粘性定位 (sticky)
  - 毛玻璃效果 (backdrop-blur)
  - 响应式菜单
  - 高亮当前页面

#### 2. ThemeToggle (主题切换)

- **功能**: 切换深色/浅色模式
- **图标**: 太阳/月亮动态切换
- **状态**: 自动同步到 localStorage

#### 3. LanguageToggle (语言切换)

- **功能**: 切换中英文
- **图标**: 地球图标
- **路由**: 自动更新 URL 路径

#### 4. MasonryGrid (瀑布流)

- **算法**: 轮询分配法
- **响应式**: 自动调整列数
- **性能**: 优化的重新渲染

#### 5. BlogCard (博客卡片)

- **信息**: 标题、摘要、日期、阅读时间、标签
- **交互**: 悬停放大效果
- **链接**: 点击跳转到文章详情

#### 6. ProjectCard (项目卡片)

- **信息**: 标题、描述、技术栈
- **按钮**: Demo 链接、源码链接
- **样式**: 统一的卡片设计

## 📄 页面结构

### 1. 首页 (`/`)

- **目的**: 自我介绍，建立第一印象
- **内容**:
  - 大标题和个人角色
  - 简短的自我介绍
  - 三个特色卡片（技能、经验、联系方式）
  - CTA 区域

### 2. 博客页面 (`/blog`)

- **布局**: 瀑布流
- **内容**: 文章卡片列表
- **功能**: 点击查看详情

### 3. 博客详情页 (`/blog/[slug]`)

- **内容**: 完整文章内容
- **元素**:
  - 返回按钮
  - 文章标题
  - 元信息（日期、阅读时间）
  - 标签
  - 文章正文

### 4. 项目页面 (`/projects`)

- **布局**: 瀑布流
- **内容**: 项目卡片列表
- **链接**: Demo 和源码链接

## 🔧 技术实现

### 1. Next.js 14 App Router

- **服务端组件**: 默认使用 RSC，提升性能
- **客户端组件**: 仅在需要交互时使用 "use client"
- **文件路由**: 基于文件系统的路由
- **中间件**: 处理 i18n 路由

### 2. TypeScript

- **类型安全**: 所有组件都有完整的类型定义
- **接口定义**: 清晰的数据结构
- **自动补全**: 提升开发体验

### 3. Tailwind CSS

- **原子化**: 使用 utility-first 方法
- **响应式**: 内置的响应式修饰符
- **深色模式**: `dark:` 前缀支持
- **自定义**: 扩展默认主题

### 4. 性能优化

- **静态生成**: 博客文章预渲染
- **代码分割**: 自动的路由级别代码分割
- **图片优化**: Next.js Image 组件（可选）
- **字体优化**: 系统字体栈

## 🎯 最佳实践

### 1. 代码组织

- **组件**: 可复用的 UI 组件
- **lib**: 工具函数和数据
- **messages**: 国际化翻译
- **app**: 页面和路由

### 2. 命名规范

- **组件**: PascalCase (如 `BlogCard`)
- **文件**: kebab-case (如 `blog-card.tsx`)
- **函数**: camelCase (如 `getBlogPosts`)

### 3. 注释规范

```typescript
/**
 * 组件/函数的简短描述
 *
 * @param param1 - 参数说明
 * @returns 返回值说明
 */
```

### 4. Git 提交规范

- `feat`: 新功能
- `fix`: 修复问题
- `docs`: 文档更新
- `style`: 代码格式
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具
